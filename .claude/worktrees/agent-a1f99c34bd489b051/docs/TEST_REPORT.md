- `GET /docs/json` serves valid OpenAPI 3.0.3 ✓
- **All 14 authenticated GETs return 401 without bearer** ✓

Bugs caught and fixed during this phase:
- Auth middleware ran at `preHandler` — AFTER schema validation. Empty POST bodies got 400 (schema error) instead of 401 (auth error), leaking schema details to unauthenticated callers. Moved to `preValidation` so auth fires first.
- `DEV_AUTH_BYPASS=1` previously *always* tried to resolve the demo workspace from DB, even with no token. With DB unreachable this returned 500 instead of 401 for anonymous traffic. Fixed: bypass only triggers when a Bearer token is present, and DB lookup is wrapped in try/catch so anonymous still gets a clean 401.

### Phase 10 — Webhook layer  ✓  **6/6 PASS**
- `POST /webhooks/clerk` valid body → reaches `prisma.user.upsert()` ✓
- `POST /webhooks/clerk` empty body → 400 `bad_body` ✓

- `POST /webhooks/music` with wrong `x-internal-secret` → 401 ✓
- `POST /webhooks/music` with correct secret → 200 ✓

Bug caught and fixed during this phase:
- The webhook plugin's `addContentTypeParser('*', ...)` was never reached for `application/json` (Fastify's default JSON parser fires first). All handlers tried to `JSON.parse((req.body as Buffer).toString())` on already-parsed objects → 400 `bad_body` on every request. Fixed: explicitly override the `application/json` parser inside the webhook plugin scope, stash raw bytes on `req.rawBody` for future signature verification, return parsed JSON on `req.body`.
